# MyWeb 内容发布流程

这份文档用于约定以后在 MyWeb 中新增文章、项目、研究笔记、生活记录的最小流程。

当前项目采用文件驱动内容管理，不使用后台 CMS。发布内容的基本方式是：新增 MDX 文件，本地预览，构建检查，提交到 Git。

## 1. 内容类型与目录

站点当前有四类内容：

* `writing`：技术文章
* `projects`：项目案例
* `research`：研究笔记
* `life`：生活记录

对应目录结构：

```text
src/content/
  writing/
    zh/
    en/
  projects/
    zh/
    en/
  research/
    zh/
    en/
  life/
    zh/
    en/
```

规则：

* 中文内容放在 `zh/`
* 英文内容放在 `en/`
* 一个文件对应一篇内容
* 文件名只作为内容文件标识，不直接决定最终 URL

## 2. URL 与双语映射规则

每篇内容至少要理解两个字段：

* `routeSlug`：控制页面 URL
* `translationKey`：控制中英文配对

示例：

```yaml
routeSlug: "agent-state-management"
translationKey: "agent-state-management"
```

对应 URL：

* 中文文章：`/writing/agent-state-management`
* 英文文章：`/en/writing/agent-state-management`

规则：

* 同一篇内容的中英文版本，`translationKey` 必须一致
* `routeSlug` 可以与文件名不同，但要保持稳定
* 如果暂时只有中文，没有英文版本，允许先发布单语内容
* 当找不到对应翻译时，语言切换会回退到目标语言的列表页，不会报错

## 3. Frontmatter 模板

### 3.1 Writing / Research / Life

这三类内容共用同一套基础字段。

```mdx
---
title: "文章标题"
routeSlug: "article-slug"
summary: "一句话摘要"
publishedAt: "2026-05-17"
updatedAt: "2026-05-18"
tags:
  - Agent
  - LLM
cover: "/images/writing/your-cover.png"
coverAlt: "cover alt"
lang: "zh-CN"
translationKey: "stable-key"
draft: false
featured: false
readTime: "6 min"
---
```

说明：

* `title`：标题，必填
* `routeSlug`：URL slug，必填
* `summary`：列表页摘要，必填
* `publishedAt`：发布日期，必填，格式建议 `YYYY-MM-DD`
* `updatedAt`：更新时间，可选
* `tags`：标签数组，可选
* `cover`：封面图路径，可选
* `coverAlt`：封面图说明，可选
* `lang`：只能是 `zh-CN` 或 `en`
* `translationKey`：中英文配对键，必填
* `draft`：`true` 时不会进入发布列表
* `featured`：当前主要用于部分首页展示控制
* `readTime`：阅读时长，仅文章 / 研究 / 生活可用

### 3.2 Projects

项目内容比普通文章多一组项目字段。

```mdx
---
title: "项目标题"
routeSlug: "project-slug"
summary: "项目摘要"
publishedAt: "2026-05-17"
tags:
  - Astro
  - MDX
cover: "/images/projects/project-cover.png"
coverAlt: "project cover"
lang: "zh-CN"
translationKey: "project-slug"
draft: false
featured: true
status: "Prototype"
stack:
  - Astro
  - TypeScript
repoUrl: "https://github.com/yourname/project"
demoUrl: "https://example.com/project"
caseStudyUrl: "https://example.com/case-study"
---
```

说明：

* `status` 必填，当前可选值：
  * `Research`
  * `Prototype`
  * `In Progress`
  * `Shipped`
  * `Archived`
* `stack`：项目技术栈数组
* `repoUrl` / `demoUrl` / `caseStudyUrl`：可选链接

## 4. 正文写法

正文直接使用 Markdown 或 MDX。

常见写法：

```md
## 背景

这里写背景。

## 方法

这里写方法。
```

如果需要插图，优先把资源放到 `public/images/` 下，再在正文中引用。

示例：

```md
![示意图](/images/writing/agent-loop.png)
```

## 5. 图片与静态资源放置规则

建议按内容类型放图：

```text
public/images/
  avatars/
  writing/
  projects/
  research/
  life/
```

规则：

* 封面图和正文图都放在 `public/images/` 下
* 路径从站点根开始写，例如 `/images/projects/demo-cover.png`
* 尽量使用稳定文件名，避免频繁改 URL

## 6. 首页取数规则

当前首页不是手工配置，而是按内容字段自动取数：

* `Featured Projects`：`projects` 中 `featured: true` 的已发布内容，按 `publishedAt` 倒序取前 3 条
* `Latest Writing`：`writing` 中已发布内容，按 `publishedAt` 倒序取前 3 条
* `Research Notes`：`research` 中已发布内容，按 `publishedAt` 倒序取前 2 条
* `Life Notes`：`life` 中已发布内容，按 `publishedAt` 倒序取前 2 条

补充：

* `draft: true` 的内容不会进入首页，也不会进入列表页
* 项目是否出现在首页精选，取决于 `featured`

## 7. 新增一篇内容的标准流程

### 7.1 新增文件

根据内容类型，在对应目录新建一个 `.md` 或 `.mdx` 文件。

例如新增中文文章：

```text
src/content/writing/zh/my-new-note.mdx
```

### 7.2 填写 frontmatter

优先保证这些字段正确：

* `title`
* `routeSlug`
* `summary`
* `publishedAt`
* `lang`
* `translationKey`
* `draft`

### 7.3 编写正文

先保证结构完整，再补图片和细节。建议最少包含：

* 背景
* 问题
* 方法 / 过程
* 结果 / 结论

### 7.4 本地预览

```bash
npm run dev
```

默认访问：

```text
http://localhost:4321
```

检查点：

* 列表页是否能看到新内容
* 详情页是否能打开
* 语言切换是否正常
* 图片路径是否正确
* 文本是否溢出或排版异常

### 7.5 构建检查

```bash
npm run build
```

如果构建失败，先修 frontmatter、图片路径、字段类型或链接格式问题。

### 7.6 提交到 Git

```bash
git add -A
git commit -m "content: add new writing note"
git push
```

如果是项目内容、研究笔记或生活记录，提交信息可以相应替换：

```text
content: add new project case study
content: add new research note
content: add new life note
```

## 8. 单语先发策略

允许先只发中文或英文，不要求每篇内容首发时就双语齐全。

推荐做法：

* 先完成中文主内容
* 英文版本后续再补
* 只要保留稳定的 `translationKey`，后续补翻译不会破坏现有路由

## 9. 发布前最小检查清单

每次新增内容前后，至少检查这几项：

* `routeSlug` 是否清晰、稳定
* `translationKey` 是否正确
* `draft` 是否为预期值
* 日期格式是否统一
* 封面图路径是否有效
* `npm run build` 是否通过

## 10. 当前建议

对你当前这个项目，最自然的维护节奏是：

* 平时直接往 `src/content/` 下增量写内容
* 先以中文为主，英文按需要补
* 项目类内容优先补 `projects`
* 技术思考优先放 `writing`
* 论文 / 实验过程优先放 `research`

这份流程文档后续如果字段、首页取数逻辑、目录约定发生变化，需要一起更新。
